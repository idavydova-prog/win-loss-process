Win/Loss Attrition Analysis — Process Automation

Strategic Brief • PACE AMER CSM Leadership • June 2026

The Problem

Every quarter, CSM leaders manually pull attrition data from Org62, classify outcomes, validate compliance against PACE guidance, and build leadership reports. This process is:

  • Time-intensive — 2–4 hours per report, with 9+ reports per quarter (3 clouds × 3 quarters)
  • Inconsistent — Classification of "Logo Loss" vs "Partial Save" varies by analyst and interpretation
  • Invisible — Compliance gaps in CSG Notes go undetected until audits, missing coaching opportunities
  • Non-scalable — Requires tribal knowledge; onboarding new leaders takes weeks of shadowing

The Solution

A standardized, automated 5-step workflow that any CSM leader can execute in minutes:

  1. Pull — Query Org62 for attrition events (configurable cloud, period, region, threshold)
  2. Classify — Apply a consistent decision tree: Risk → Actionable → Win / Logo Loss / Partial Save / SP Downgrade / Not Actionable
  3. Validate — Check every record against PACE hashtag guidance (license tags, sig tags, date stamps)
  4. Report — Generate interactive HTML with charts, drill-downs, compliance tables, and Org62 links
  5. Verify — 3-pass automated validation for classification accuracy and report integrity

Invocation: Any team member types: "Run the Win/Loss Process for [Cloud] for [Period]" — and the full 5-step process executes end-to-end.
